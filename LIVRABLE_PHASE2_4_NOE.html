H-Compta AI — Authentification + HTML connectés

Vos 4 pages sont maintenant branchées sur Supabase. La connexion est réelle, les données sont dynamiques, et la sécurité est en place.

🔐 Authentification — Ce qui a changé
🐸 En langage simple
Avant, la connexion était simulée : n'importe qui pouvait se connecter avec les comptes test hardcodés. Maintenant c'est Supabase Auth — la même technologie que les apps bancaires. Chaque utilisateur a un vrai compte, un vrai mot de passe, et un vrai token JWT.
✅ Ce qui fonctionne maintenant
  • Connexion réelle — email + mot de passe vérifiés par Supabase Auth
  • 4 rôles — PME, Expert, Collaborateur, Admin (chacun voit ses données)
  • Redirection automatique — PME → Dashboard PME, Expert → Dashboard Expert
  • Réinitialisation mot de passe — email natif Supabase (plus besoin de Make S7)
  • Déconnexion sécurisée — session invalidée côté serveur
  • Isolation PME — pme@test.ci ne voit QUE CimenCI SA (testé et prouvé)

📄 Les 4 pages HTML — Ce qui a changé
🐸 En langage simple
Vos 4 pages sont visuellement identiques. On n'a touché à aucun pixel. Ce qui a changé c'est le moteur sous le capot : toutes les données viennent de Supabase, toutes les actions écrivent dans Supabase, et Mariah passe par un proxy sécurisé.
PageAvantAprès
Login (login.html) SIMULATION_MODE = true, comptes hardcodés Supabase Auth, vrais comptes, JWT
Dashboard PME (dashboard-pme.html) KPIs hardcodés, import simulé, Mariah sans clé API KPIs depuis Supabase, upload Storage, Mariah via proxy sécurisé
Dashboard Expert (dashboard-expert.html) 5 PME hardcodées, Mariah = réponses cycliques, validation DOM-only Portefeuille dynamique, Mariah via proxy, validation écrit en BDD
Landing (index.html) Liens vers anciens fichiers HTML Liens corrigés (login.html, dashboard-pme.html)
🔒 Sécurité — Ce qui est protégé
  • ✅ Clé API Claude (Mariah) — JAMAIS dans le navigateur. Passe par un proxy serveur (Edge Function).
  • ✅ Données PME — Isolées par RLS. Chaque PME ne voit que ses données. Testé en production.
  • ✅ Sessions — JWT Supabase (1h + refresh automatique). Pas de localStorage.
  • ✅ Mot de passe — Hashé par Supabase. Jamais stocké en clair.
  • ✅ Clé service_role — Uniquement côté serveur (Make + Edge Function). JAMAIS côté client.
